# Body data — rotation marker + atmosphere gradient + description

**Date:** 2026-05-14
**Validation:** ⏳ pending — landed and tested in CI (full suite green,
lint + typecheck clean, two new OrbitDiagram tests added). Not yet
exercised against a live KSP session.

## What changed

Wired the unused `b.*` body keys into OrbitView and SystemView per the
2026-05-14 audit follow-up. All keys were already on the wire from
the Telemachus fork; only schema, meta, hook fan-out, and widget
consumption were missing.

### Schema additions

Eight new indexed `b.*` keys in `TelemaachusSchema`:

- `b.atmosphereContainsOxygen` (bool)
- `b.hillSphere`, `b.mass`, `b.geeASL` (numbers)
- `b.rotationPeriod`, `b.rotationAngle`, `b.rotates` (numbers + bool)
- `b.tidallyLocked`, `b.ocean` (bools)
- `b.description` (string)

### `useCelestialBodies` extension

Six new entries on the BODY_FIELDS fan-out — description, hillSphere,
rotates — sit alongside the existing fields. They're effectively
static per body (Telemachus emits them once after subscribe), so the
extra subscribe load is negligible.

`b.rotationAngle` was *not* added to the system-wide fan-out. It ticks
at the WS frame rate (~4 Hz × 17 bodies = 68 bumps/s) — gated behind
a new `useBodyRotation(bodyName)` hook that subscribes to a single
body at a time. OrbitView is the only consumer; SystemView doesn't
need the live angle.

### OrbitDiagram opt-in props

Three new optional props plumbed through the shared `OrbitDiagram`:

- `rotationAngleDeg?: number | null` — when set, renders a thin
  diameter line + small dot at the rotating "near-side meridian"
  position. Only in the "full" variant (mini frames are too small for
  a useful read).
- `atmosphereDepthM?: number | null` — soft radial-gradient circle
  from body surface to `bodyRadius + atmosphereDepthM`. Drawn before
  the body disc so the solid fill occludes the inner edge.
- `atmosphereHasOxygen?: boolean | null` — blue tint when true,
  amber when false.

Defaults are `null`, so existing consumers (`ManeuverPreview`,
`CurrentOrbit`, `OrbitDiagram.test.tsx`) render unchanged. Two new
tests verify the opt-in contract: marker omitted by default, present
when prop set; atmosphere band same.

### OrbitView wiring

Calls `useBodyRotation(bodyName)` for the live angle. Passes:

- `rotationAngleDeg`: `null` if `b.rotates === false`, else the angle.
- `atmosphereDepthM`: `body.maxAtmosphere` from the static body
  registry — Kerbol stock bodies already carry this.
- `atmosphereHasOxygen`: `true` for `Kerbin` / `Laythe`, `false`
  otherwise. Cheap heuristic; replaced when the static body registry
  grows an explicit `hasOxygen` field.

### SystemView AlmanacPanel

Three new optional rows in `buildRows`:

- **Hill sphere** (when present) — sits next to the SOI row.
- **"Does not rotate"** — added for non-rotating bodies (notably
  tidally-locked Mun-like bodies that nonetheless tick a
  `rotationPeriod === orbital period`).
- **Body description** — KSP's per-body flavour text rendered as a
  paragraph below the stats grid. Scroll handles overflow.

## Why no rotation interpolation

The follow-ups doc raised "interpolate between samples since Telemachus
only ticks at WS rate" as an open question. Skipped on first cut: at
1× warp the per-tick rotation delta on Kerbin is 0.004° — well below
visible. Under high warp it'll step but the operator is already
watching the wider time-warp UI for cues. Revisit if the marker reads
as jittery in practice.

## Validation checklist (next live session)

- OrbitView during Kerbin orbit: atmosphere band visible in blue,
  rotation marker traces the body limb at the day-length cadence.
- OrbitView during Eve orbit: atmosphere band visible in amber, no
  oxygen tint.
- OrbitView during Mun orbit: no atmosphere band, marker still
  rotates (Mun rotates; tidally-locked vs not visible via
  `b.rotates`).
- Time-warp to 10000× and watch the rotation marker — confirm it
  doesn't read as jittery. If it does, revisit interpolation.
- SystemView almanac on a body with a long description (Mun's a good
  one — has multiple paragraphs of flavour text) — text wraps
  cleanly, scroll handles long copy at smaller widget sizes.
- AlmanacPanel hill-sphere row appears between SOI and orbital
  period.

## What didn't ship

- The doc also mentioned `b.timeWarpAltitudeLimits` and
  `b.orbitingBodies` / `b.position`. Those weren't part of the
  rotation / gradient / description slice — they can land later if
  needed.
- Static body registry doesn't carry a `hasOxygen` flag yet —
  hard-coded "Kerbin / Laythe = oxygen" instead. Trivially replaced
  later.
